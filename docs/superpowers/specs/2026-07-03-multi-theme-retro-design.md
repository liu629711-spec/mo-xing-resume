# 多主题简历网站 —— 水墨长卷 × 复古电脑（像素房间）设计规格

> 版本：2026-07-03  
> 状态：待用户审查  
> 定位：游戏产品运营求职简历 · 双风格可切换 · CMS 分主题独立内容

---

## 1. 设计目标

在现有**水墨长卷 3D 展卷**基础上，新增第二主题 **「复古电脑 / 进入游戏世界」**。访客可在站内切换风格；两套主题**入场动画互不影响**，**内容数据完全独立**，后台按主题分别管理。

**核心体验（复古主题）**：

> 第一人称走进一间像素 3D 小房间，桌上是电脑。走近、低头敲 Enter，显示器亮起，镜头吸入屏幕——进入「电脑里的游戏世界」，随探索感受游戏/电脑从起源到现代的演进。

**记忆点检验**：访客能复述「我进了一个满是游戏元素的小房间，坐下去敲了回车，钻进了屏幕里」。

## 4. 双主题架构

```
                         ┌──────────────────┐
访客 ──────────────────►│  Theme Manager   │
                         └────────┬─────────┘
                    ┌─────────────┴─────────────┐
                    ▼                           ▼
            ┌───────────────┐           ┌───────────────┐
            │  ink 水墨     │           │  retro 复古   │
            │  boot: 墨滴开卷│           │  boot: 像素房间│
            │  3D 长卷山水   │           │  房间3D→屏内2D │
            └───────┬───────┘           └───────┬───────┘
                    │                           │
                    └───────────┬───────────────┘
                                ▼
                    data/themes/{themeId}/…
                    （内容完全独立）
```

### 4.1 切换行为

- **切换器**：全局 UI（水墨=右上角；复古=任务栏/桌面 Widget），显示当前主题名。
- **切换流程**：淡出 → 卸载当前主题 runtime → 加载新主题 → 执行**该主题专属 boot**（水墨墨滴 / 复古房间，互不混用）。
- **持久化**：`localStorage.theme`；可选 URL `?theme=retro|ink` 分享不同风格。
- **跳过**：两主题 boot 均支持 Esc /「跳过」→ 直接进入主界面（水墨跳过开卷 / 复古跳过房间直达桌面）。

### 4.2 技术分层

| 层 | 职责 |
|----|------|
| `js/core/theme-manager.js` | 注册主题、切换、boot 调度 |
| `js/core/data-loader.js` | 按 `activeTheme` 加载 `data/themes/{id}/` |
| `js/themes/ink/` | 现有水墨逻辑迁入（three、sections、effects） |
| `js/themes/retro/` | 房间 scene、boot、desktop、windows、eras |
| `js/shared/` | draft、video、schema 工具等 |

## 5. 复古主题：第一人称像素房间（Boot 分镜）

**视觉**：3D 低模块面 + **像素化后处理**（低分辨率 render + nearest 放大 + CRT 扫描线/色偏/vignette）。整体气质参考像素 FPS / SUPERHOT 式方块空间，而非写实房间。

### 5.1 分镜（约 15–25s，可跳过）

| 阶段 | 时长 | 画面 | 交互 |
|------|------|------|------|
| 0 入场 | 0enic | 像素 3D 小房间全景；环境暗，台灯/显示器待机微光 | — |
| 1 走近 | 3–5s | 第一人称/轨道相机移向书桌（键盘、鼠标、主机、显示器、椅） | 可自动，或 WASD 短距移动（P2 可选） |
| 2 就位 | 1s | 停在键盘前，视角略俯 | — |
| 3 低头 | 1s | 相机 pitch 向下，Enter 键高亮 + 提示「按 Enter 开始」 | 等待 Enter |
| 4 敲击 | 0.5s | 键帽下压动画 + 8-bit 按键音；主机电源灯亮 | Enter |
| 5 抬头 | 1s | 相机 pitch 回正，显示器由暗到亮（扫描线、开机画面） | — |
| 6 吸入 | 1–2s | 镜头向屏幕推进，边缘畸变 + 像素块化转场 + 白闪 | — |
| 7 进入 | — | 屏内：BIOS 一闪或直接进入复古桌面（P1 可简化） | — |

### 5.2 房间内游戏代表元素（环境叙事）

用户确认：**房间内可出现各游戏的代表元素**，作为「游戏史博物馆 / 玩家房间」布景，不喧宾夺主。

**布置原则**：

- **密度**：桌面为主舞台，墙面/书架/地面为辅；总元素 8–15 个，避免 clutter。
- **尺度**：像素块面道具，与房间统一；不做高精度模型。
- **互动**：P1 仅 hover 高亮 + tooltip（游戏名 + 一句关联）；P3 可点击弹出「彩蛋卡片」。
- **版权**：使用用户认可的**真实游戏代表素材**（图标、道具剪影、卡带盒、海报像素化等）；页脚统一免责声明（见 §10）。

**建议分区与示例元素**（实现时按素材可得性调整，均为**致敬/代表物**，非完整复制）：

| 区域 | 代表元素示例 | 叙事含义 |
|------|-------------|----------|
| 显示器上方 | 像素海报：马里奥问号块 / 三角力量 | 起源：FC / 任天堂 |
| 书架 | 卡带盒 spine：FF、DQ、塞尔达（像素化封面） | 16-bit RPG 时代 |
| 桌面左侧 | 红色胶囊 / 绿色水管道具 | 平台动作 |
| 桌面右侧 | 药水瓶、技能球小摆件 | ARPG |
| 主机旁 | 旧式软盘 / CD 盒（Diablo / CS 时代） | PC 网游萌芽 |
| 墙上 | 街机摇杆剪影 / 像素剑 | 街机 & 动作 |
| 地面 | 掌机像素模型 | 便携时代 |
| 椅背 | 手游图标贴纸（抽象化） | 现代运营 |

**与简历映射**：元素按**年代由远及近**布置（左/上=老，右/下=新），与屏内 UI 年代递进一致。

### 5.3 屏内世界（进入显示器之后）

沿用参考2 结构，并按**游戏/电脑演进**组织：

| 顺序 | 屏内界面 | 致敬气质 | 对应内容类型 |
|------|----------|----------|--------------|
| 1 | 复古桌面 + 图标 | Win95 / DOS | 导航枢纽 |
| 2 | 关于 | 8-bit 角色卡 / FC HUD | profile |
| 3 | 技能 | ARPG 技能树 | skills |
| 4 | 项目 | FF 列表 → 横版关卡详情 | projects |
| 5 | 数据 | 塞尔达式成就符文 | metrics |
| 6 | 碑石 | 卡带架 / 游戏库 | games |
| 7 | 经历 | DQ 文字框翻页 | timeline |
| 8 | 联系 | 组队招募对话框 | contact |
| 9 | 结算 | 通关评价画面 | 探索度 + 联系 CTA |

**年代递进策略（用户倾向）**：一次访问中，越深入板块，UI 越「现代」——从 8-bit → 16-bit 菜单 → PS 水晶 UI → 现代扁平，隐喻「电脑和游戏一起变好」。

## 6. 水墨主题（现有）

- **保持不变**：墨滴开卷、Three.js 长卷、七大景致、四季、CMS 现有字段。
- **迁移**：代码迁入 `js/themes/ink/`，数据迁入 `data/themes/ink/`。
- **切换**：离开水墨时卸载 Three 场景；进入时重新 boot，不保留 retro 状态。

## 7. 内容模型（CMS · 完全独立）

用户确认：**水墨与复古两套 JSON 互不影响**。

### 7.1 目录

```
data/
├── themes.json              # 主题清单：id, label, defaultBoot, cmsLabel
└── themes/
    ├── ink/                 # 现有结构平移
    │   ├── profile.json
    │   ├── skills/ …
    │   └── …
    └── retro/               # 独立字段，可含游戏化文案
        ├── profile.json
        ├── skills/ …
        └── …
```

### 7.2 后台 UI

- 顶部或侧边 **主题切换**：`水墨长卷` | `复古电脑`。
- Collections 按主题分组，路径前缀 `data/themes/ink/` vs `data/themes/retro/`。
- 复古主题**额外字段示例**（实现时在 config.yml 定义）：

| 集合 | 复古专有字段 |
|------|-------------|
| profile | `className`（职业）、`level`、`titleScreen`（标题屏文案） |
| skills | `branch`（天赋系）、`skillPointCost` |
| projects | `dungeonName`、`bossName`、`clearRate`（隐喻通关率） |
| games | `platform`（平台）、`cartridgeColor` |
| timeline | `chapterTitle`、`dialogue`（DQ 式对话） |
| metrics | `achievementId`、`rarity` |

水墨 collections **字段不变**。

## 10. 版权与免责声明

用户要求：可使用真实游戏代表素材；页脚增加侵权联系删除。

**建议页脚文案**（两主题共用，retro 主题更显眼）：

> 本站部分界面与素材致敬经典游戏作品，相关版权归原权利人所有。如有侵权，请联系 [邮箱] 删除。

**实现建议**：

- 房间内/屏内素材尽量**低分辨率像素化**，注明致敬来源（tooltip 或「关于本站」页列表）。
- 不使用完整 ROM、原声大段 BGM 等高风险素材；音效优先自制或 CC0。
- 免责声明不替代授权，体现善意配合删除。

## 11. 性能与适配

| 场景 | 策略 |
|------|------|
| 桌面 | 房间 3D + 像素后处理；屏内 2D DOM + 轻量 Canvas |
| 平板 | 降低 render 分辨率；简化房间元素数量 |
| 手机 | retro boot 简化为 2D 插画版或直接进入桌面；ink 仍 2D 降级 |
| reduced-motion | 跳过房间相机动画，直接显示桌面 |
