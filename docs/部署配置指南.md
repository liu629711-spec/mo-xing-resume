# 部署配置指南 · 墨行简历后台

本指南引导你把项目部署到 Netlify 并开启 Decap CMS 后台，实现可视化编辑内容、保存即自动上线。

## 前置条件

- GitHub 账号
- Netlify 账号（可用 GitHub 登录）

## 步骤 1：项目推到 GitHub

1. 在 GitHub 新建仓库（建议私有：`mo-xing-resume`）
2. 本地项目初始化并推送：

   ```bash
   git init
   git add .
   git commit -m "init: 墨行个人介绍网站"
   git branch -M main
   git remote add origin git@github.com:<你的用户名>/mo-xing-resume.git
   git push -u origin main
   ```

> 注意：`admin/config.yml` 里 `backend.branch` 默认是 `main`，若你的默认分支是 `master`，需把 config.yml 里的 `branch: main` 改成 `branch: master`。

## 步骤 2：Netlify 连接仓库

1. 登录 https://app.netlify.com
2. "Add new site" → "Import an existing project" → 选 GitHub → 选 `mo-xing-resume` 仓库
3. 构建配置（应自动读取项目根目录的 `netlify.toml`）：
   - Build command: `node scripts/gen-index.mjs`
   - Publish directory: `.`
4. 点 Deploy，等待首次部署完成

> 构建脚本会扫描 `data/skills`、`data/projects`、`data/games`、`data/timeline` 目录，自动生成各目录的 `index.json` 文件列表，供前端聚合加载。

## 步骤 3：开启 Identity（登录系统）

1. Netlify 站点后台 → "Identity" 标签 → "Enable Identity"
2. Registration preferences：选 **"Invite only"**（只允许被邀请的人，即你自己）
3. 添加你自己：点 "Invite users" → 填你的邮箱 → 收邮件激活并设密码

## 步骤 4：开启 Git Gateway（让后台能提交到仓库）

1. Netlify 站点后台 → "Identity" → "Services" → "Git Gateway" → "Enable"
2. 授权 Netlify 访问你的 GitHub 仓库

## 步骤 5：进入后台

1. 访问 `https://<你的站点>.netlify.app/admin/`
2. 用步骤 3 激活的账号登录
3. 看到 7 个板块集合：
   - **关于 · 个人介绍**（单文件）
   - **技艺 · 技能矩阵**（可增删分类）
   - **项目 · 案例经历**（可增删项目）
   - **数据成河 · 数据看板**（单文件）
   - **运营碑石 · 游戏作品**（可增删碑石，含视频/富文本/指标）
   - **时间轴 · 经历**（可增删节点）
   - **联系 · 联系方式**（单文件）
4. 点进去可视化表单编辑
5. 改完点保存 → 自动 git commit → 触发 Netlify 重新构建 → 30-60 秒后线上生效

## 运营碑石视频配置

运营碑石支持演示视频，在碑石编辑页的"演示视频链接"字段填链接，前端自动识别：

- **mp4 / webm 直链**：用 `<video>` 标签直接播放（推荐）
- **B 站视频页链接**：用 iframe 嵌入
- **YouTube 链接**：用 iframe 嵌入

### 视频存储建议

录屏文件较大（几十到几百 MB），**不建议传仓库**（GitHub 单文件建议 <50MB）。推荐：

- **阿里云 OSS / 腾讯云 COS**（最推荐）：传 mp4 拿直链，填到"演示视频链接"字段。费用极低（几毛/月量级），无广告，国内访问稳定，最适合理简历场景。
- **B 站**：传后填视频页链接（有广告和推荐视频，不太适合理简历场景）
- **YouTube**：国内访问不稳定，不推荐

封面图字段可上传到仓库 `uploads/` 目录或填外部图片 URL。

## 草稿/隐藏功能

每个可增删的条目（技艺/项目/碑石/时间轴）都有"草稿(隐藏)"开关：

- 开启后该条目在前端不显示（不会渲染），但数据仍在
- 方便你"快速拿掉某条"而不真删，关掉开关即恢复显示

## 常见问题

**进不了后台**
- 确认 Identity 已开启
- 确认你的账号已通过邮件激活
- 确认 Git Gateway 已启用并授权

**保存后线上没更新**
- Netlify 构建需 30-60 秒，查看 Netlify 控制台 "Deploys" 是否在构建中
- 若构建失败，查看构建日志（可能是 `node scripts/gen-index.mjs` 报错，检查 data 目录结构）

**草稿条目不显示**
- 开了"草稿(隐藏)"开关的条目前端不渲染，关掉开关即恢复

**某板块内容不显示**
- 检查对应 `data/<name>/index.json` 是否包含该文件名（构建期自动生成，正常无需手动维护）
- 检查 `data/<name>/<slug>.json` 是否符合 schema（必填字段缺失会被校验拦截，回退占位）

**本地开发**
- 运行 `node scripts/gen-index.mjs` 生成 index.json
- 运行 `npm run dev`（npx serve .）本地预览
- 运行 `npm test` 跑单元测试
